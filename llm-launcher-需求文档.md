# LLM 本地模型启动器 — 需求文档

> 目标：一个简易桌面应用，用于可视化加载本地 GGUF 模型、自定义运行参数、后台检测与进程管理。
> 开发策略：阶梯制，每阶段确认可行后再进入下一阶段。

---

## 技术选型建议

| 方案 | 优点 | 缺点 |
|------|------|------|
| **Python + Textual** | 终端 TUI，跨平台，开发快 | 不是图形窗口，视觉受限 |
| **Python + PySide6** | 原生桌面控件，跨平台 | 代码量偏大 |
| **Python + PyWebView** | HTML/CSS 写界面，Python 写逻辑 | 依赖浏览器内核 |
| **Electron + React** | UI 效果最好，生态丰富 | 打包体积大（~200MB） |

> **推荐第一阶段用 Textual**（终端即界面，零打包成本，专注逻辑验证）。第二阶段再换 GUI。

---

## 第一阶段：核心启动控制（MVP）

### 目标
替代现有的 `.bat` 菜单脚本，实现可视化参数配置 + 一键启动/停止。

### 功能需求

#### 1.1 模型选择
- [x] 浏览本地文件选择 `.gguf` 主模型（`FileBrowser` 弹窗）
- [x] 浏览本地文件选择 `mmproj` 视觉投影文件（可选，独立浏览按钮）
- [x] 记住上次选择的路径（`ConfigStore` 写入 `config.yaml`）
- [x] 自动检测同目录下的 mmproj（文件名含 `mmproj` 且为 `.gguf`）

#### 1.2 核心参数配置
- [x] 上下文大小（`-c`）：下拉 2048 / 4096 / 8192 / 16384 / 32768
- [x] GPU 层数（`--n-gpu-layers`）：数字输入 + 「全GPU」(9999) / 「CPU」(0) 快捷按钮
- [x] 并发数（`-np`）：下拉 1 / 2 / 4 / 8
- [x] 端口（`--port`）：数字输入框，默认 8080，冲突时提示（`PortInUseError`）
- [x] 监听地址（`--host`）：下拉 127.0.0.1（仅本机）/ 0.0.0.0（局域网）

#### 1.3 服务器生命周期
- [x] **启动**按钮：调用 `ProcessSupervisor.start()` 启动 llama-server
- [x] **停止**按钮：SIGTERM → 等 3s → SIGKILL
- [x] 状态指示：运行中（绿）/ 已停止（灰）/ 启动中（黄）/ 异常退出（红）
- [x] 启动后自动打开浏览器（`auto_open_browser` Switch 开关）

#### 1.4 进程存活检测
- [x] 实时检测 llama-server 进程是否在运行（`_poll_loop` 每 1 秒轮询 PID）
- [x] 进程意外退出时弹通知 + 自动回收到"已停止"状态（`crashed` 状态 → `notify` 弹窗）
- [x] 端口被占用时提示用户换端口（`PortInUseError` 捕获，弹窗提示）

#### 1.5 基础日志
- [x] 显示 llama-server 的标准输出/错误输出（只读，`RichLog` 面板，stderr 已合并到 stdout）
- [x] 日志自动滚动到最新行
- [x] 关键事件高亮（加载完成 / 错误 / OOM，`add_line()` 中关键词染色：绿/红/黄）

### 验收标准
- [x] 能通过 GUI 选择模型文件、调整 5 个核心参数、启动/停止服务器
- [x] 启动后浏览器能正常访问 Web UI 和 API（手动测试验证通过）
- [x] 进程异常退出时 UI 状态正确更新

## 第二阶段：高级参数与运行时监控

### 目标
覆盖更多 llama.cpp 参数，加入实时资源监控。

### 功能需求

#### 2.1 高级参数

在控制面板新增「高级参数」折叠区（Textual `Collapsible` 组件），按类别分组：

**KV Cache 与显存**
- [ ] **KV Cache 量化类型**（`-ctk` / `-ctv`）：下拉 f16 / q8_0 / q4_0，默认 f16
- [ ] **统一 KV 池**（`-kvu` / `--kv-unified`）：Switch 开关，多 slot 共享 KV
- [ ] **KV 不放 GPU**（`--no-kv-offload`）：Switch 开关，省显存
- [ ] **Flash Attention**（`-fa`）：Switch 开关，省显存加速
- [ ] **Prompt Cache**（`--cache-prompt`）：Switch 开关，缓存相同前缀
- [ ] **空闲 Slot 复活**（`--cache-idle-slots`）：Switch 开关
- [ ] **Cache RAM 上限**（`--cache-ram`）：数字输入，默认 8192 MiB

**推理速度**
- [ ] **线程数**（`-t`）：数字输入，默认 -1（CPU 核心数）
- [ ] **Prompt 线程数**（`-tb`）：数字输入，默认等于 `-t`
- [ ] **逻辑批大小**（`-b`）：数字输入，默认 2048
- [ ] **物理批大小**（`-ub`）：数字输入，默认 512
- [ ] **HTTP 线程数**（`--threads-http`）：数字输入，默认 -1
- [ ] **跳过预热**（`--no-warmup`）：Switch 开关

**采样参数**
- [ ] **温度**（`--temp`）：滑块 0.0 ~ 2.0，默认 0.80
- [ ] **Top-K**（`--top-k`）：数字输入，默认 40
- [ ] **Top-P**（`--top-p`）：滑块 0.0 ~ 1.0，默认 0.95
- [ ] **Min-P**（`--min-p`）：滑块 0.0 ~ 1.0，默认 0.05
- [ ] **重复惩罚**（`--repeat-penalty`）：滑块 0.5 ~ 2.0，默认 1.0
- [ ] **随机种子**（`-s`）：数字输入，-1 为随机
- [ ] **最大生成长度**（`-n`）：数字输入，-1 为无限
- [ ] **忽略 EOS**（`--ignore-eos`）：Switch 开关，忽略结束符持续生成

**思考/推理模式**
- [ ] **思考模式**（`-rea` / `--reasoning`）：下拉 on / off / auto
- [ ] **思考格式**（`--reasoning-format`）：下拉 none / deepseek / deepseek-legacy
- [ ] **思考预算**（`--reasoning-budget`）：数字输入，-1 不限，0 不思考

**多模态**
- [ ] **视觉编码器放 GPU**（`--mmproj-offload`）：Switch 开关，默认开启
- [ ] **最小视觉 Token**（`--image-min-tokens`）：数字输入
- [ ] **最大视觉 Token**（`--image-max-tokens`）：数字输入

**安全与访问控制**
- [ ] **API Key**（`--api-key`）：文本输入，留空不传参
- [ ] **超时秒数**（`--timeout`）：数字输入，默认 600
- [ ] **Prometheus 监控**（`--metrics`）：Switch 开关
- [ ] **Slots 端点**（`--slots`）：Switch 开关，默认开启

实现要点：
- `_build_command` 中对上述参数做条件拼接（空值/默认值不传参，避免干扰 llama-server 默认行为）
- 所有参数写入 `ConfigStore` 并在启动时读取回填
- 参数分组使用 `Collapsible` 折叠，默认展开常用组（KV Cache、采样参数）

#### 2.2 运行时监控
- [ ] GPU 利用率 / 显存占用（调用 nvidia-smi）
- [ ] 进程内存占用（RSS）
- [ ] 当前活跃请求数（轮询 `/slots` 端点）
- [ ] 每 slot 的上下文使用情况

#### 2.3 预设管理
- [ ] 保存当前参数为预设（命名 + 存储）
- [ ] 预设列表，一键切换
- [ ] 预设导入/导出（JSON 文件）
- [ ] 覆盖预设中的已有名称时二次确认

#### 2.4 API 测试面板
- [ ] 内建简易聊天框，直接测试 API（纯文本）
- [ ] 支持粘贴图片 URL 测试多模态

### 验收标准
- [ ] 高级参数全部可用，与 llama.cpp 命令行效果一致
- [ ] 参数分组折叠显示，默认展开常用组
- [ ] 空值/默认值不传参，避免干扰 llama-server 默认行为
- [ ] 监控面板正确显示 GPU 占用和请求状态
- [ ] 预设可保存、切换、导出、导入
n#### 延迟参数（待正式 UI 后实现）
- `--grammar` / `--json-schema`：约束输出格式（需文本输入区 UI）
- `--reasoning-budget-message`：推理预算用完时注入的提示语

## 第三阶段：多模型管理与自动化

### 目标
支持多模型切换、模型下载、开机自启等。

### 功能需求

#### 3.1 多模型管理
- [ ] 模型库列表（扫描指定目录下的所有 GGUF）
- [ ] 每个模型记忆各自的参数预设
- [ ] 模型信息展示（参数量、量化类型、文件大小）
- [ ] 一键切换模型（先停旧、再启新）

#### 3.2 远程模型下载
- [ ] 输入 HuggingFace repo ID，自动下载 GGUF + mmproj
- [ ] 下载进度条
- [ ] 断点续传提示

#### 3.3 自动化
- [ ] 开机自启选项（写 Windows 启动项或任务计划）
- [ ] 崩溃自动重启（可配置重试次数和间隔）
- [ ] 空闲自动休眠（`--sleep-idle-seconds`）

#### 3.4 托盘模式
- [ ] 最小化到系统托盘
- [ ] 托盘图标显示运行状态（绿/灰）
- [ ] 右键菜单：启动/停止/退出

### 验收标准
- [ ] 多模型切换流畅，不丢参数
- [ ] 远程下载模型成功
- [ ] 托盘模式可用

## 第四阶段：用户体验优化

### 功能需求

#### 4.1 首次使用引导
- [ ] 检测 llama.cpp 是否安装，未安装时引导下载
- [ ] 首次使用走一遍 3 步引导：选模型 → 调参数 → 启动

#### 4.2 错误诊断
- [ ] 常见错误自动识别并给出中文建议
  - `unknown projector type` → llama.cpp 版本太旧
  - `CUDA error` → 驱动不兼容，建议降级/升级
  - `out of memory` → 建议减上下文或换量化版本
- [ ] 一键复制诊断信息用于反馈

#### 4.3 国际化与可访问性
- [ ] 界面语言中/英切换
- [ ] 页面字体大小可调

## 与现有脚本的关系

现有的 `启动 MiniCPM.bat` 和 `start-minicpm.ps1` 作为轻量替代方案继续保留。
桌面应用不替代它们，而是提供另一条路径——**脚本给快速用，桌面应用给日常用**。
