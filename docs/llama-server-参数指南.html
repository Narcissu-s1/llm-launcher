模型加载

最常用的方式是 -m 指定本地路径，或 -hf 从 HuggingFace 自动下载。
-m/--model FNAME
LLAMA_ARG_MODEL
指定要加载的模型文件路径（.gguf 格式）。这是最基本也最常用的参数。
-mu/--model-url URL
LLAMA_ARG_MODEL_URL
从 URL 直接下载模型文件。
-hf/--hf-repo <user>/<model>[:quant]
LLAMA_ARG_HF_REPO
从 HuggingFace 仓库下载模型。quant 可选，默认 Q4_K_M。如果仓库有 mmproj 文件会自动下载。
示例：ggml-org/GLM-4.7-Flash-GGUF:Q4_K_M
-hff/--hf-file FILE
LLAMA_ARG_HF_FILE
指定 HuggingFace 仓库中的具体文件名，会覆盖 --hf-repo 中的 quant 选择。
-hft/--hf-token TOKEN
HF_TOKEN
HuggingFace 访问令牌，用于下载私有或受限模型。
-dr/--docker-repo [repo/]model[:quant]
LLAMA_ARG_DOCKER_REPO
从 Docker Hub 模型仓库获取模型。默认仓库前缀 ai/。示例：gemma3
-mv/--model-vocoder FNAME
声码器模型路径，用于 TTS 音频生成。
--offline
LLAMA_OFFLINE
离线模式：强制使用缓存，阻止一切网络访问。

上下文与批处理

上下文越大显存占用越高。批处理大小影响 prompt 处理速度，但不影响生成速度。
-c/--ctx-size N
默认: 0 (从模型读取)LLAMA_ARG_CTX_SIZE
上下文窗口大小，即模型一次能处理的最大 token 数量。设 0 表示使用模型训练时的默认值。常用值：4096、8192、16384、32768。值越大消耗显存越多。
-n/--n-predict N
默认: -1 (无限)LLAMA_ARG_N_PREDICT
最大生成长度。-1 表示不限制，直到模型自然结束。设为正整数可控制最多生成多少 token。
-b/--batch-size N
默认: 2048LLAMA_ARG_BATCH
逻辑批大小。控制 prompt 处理阶段一次处理多少 token。增大可加速长 prompt 处理，但会增加显存峰值。
-ub/--ubatch-size N
默认: 512LLAMA_ARG_UBATCH
物理批大小。将逻辑批再分割为更小的物理批，在 GPU 上实际执行。通常不需要修改。
--keep N
默认: 0
上下文移位时保留的初始 prompt token 数量。-1 表示保留全部。仅在交互/无限文本生成模式下有意义。
-np/--parallel N
默认: -1 (自动)LLAMA_ARG_N_PARALLEL
并发 slot 数。决定服务器能同时处理多少个请求。每个 slot 会占用独立的 KV cache 显存（除非启用统一 KV）。
--context-shift/--no-context-shift
默认: 禁用LLAMA_ARG_CONTEXT_SHIFT
无限文本生成时的上下文移位。启用后当上下文写满时自动丢弃早期内容继续生成。
-cb/--cont-batching
默认: 启用LLAMA_ARG_CONT_BATCHING
连续批处理（动态批处理）。允许多个请求共享同一批次，显著提升并发吞吐量。几乎没有理由禁用。

CPU 线程与调度

大多数情况下保持默认即可。只有 CPU 推理或混合推理时需要精细调节。
-t/--threads N
默认: -1 (自动)LLAMA_ARG_THREADS
生成阶段的 CPU 线程数。-1 表示自动检测 CPU 核心数。如果用纯 GPU 推理，此参数影响较小。
-tb/--threads-batch N
默认: 同 --threads
批处理/prompt 处理阶段的线程数。prompt 处理可并行度更高，可设为比 -t 更大的值。
--threads-http N
默认: -1LLAMA_ARG_THREADS_HTTP
处理 HTTP 请求的线程数。-1 为自动。高并发场景可适当增加。
--prio N
默认: 0 (normal)
设置进程/线程优先级。
-1 low0 normal1 medium2 high3 realtime
-C/--cpu-mask M
默认: "" (不限制)
CPU 亲和性掩码（十六进制），控制推理线程运行在哪些 CPU 核心上。与 --cpu-range 互补。
-Cr/--cpu-range lo-hi
指定 CPU 核心范围，如 0-7。与 --cpu-mask 互补使用。
--cpu-strict <0|1>
默认: 0
是否使用严格 CPU 绑定。设为 1 时线程不会被调度到其他核心。
--poll <0...100>
默认: 50
轮询等级。0 = 不轮询（省 CPU），100 = 最大轮询（最低延迟但占满 CPU）。
--numa TYPE
NUMA 优化策略（多路服务器）。
distributeisolatenumactl

GPU 与多卡

单卡用户通常只需设置 -ngl 即可。多卡用户需关注 -sm 和 -ts。
-ngl/--n-gpu-layers N
默认: autoLLAMA_ARG_N_GPU_LAYERS
卸载到 GPU 的模型层数。auto 自动适配显存；设为一个大数（如 9999）或 all 表示全部放 GPU；设为 0 表示纯 CPU 推理。
autoall0 (CPU)数字
-dev/--device <dev1,dev2,...>
LLAMA_ARG_DEVICE
指定用于卸载的设备列表。none = 不卸载。用 --list-devices 查看可用设备。
-sm/--split-mode {none,layer,row,tensor}
默认: layerLLAMA_ARG_SPLIT_MODE
多 GPU 拆分策略。
none 仅用一张卡 layer 按层拆分 (流水线) row 按行拆分 (并行) tensor 按张量拆分 (实验)
-ts/--tensor-split N0,N1,...
LLAMA_ARG_TENSOR_SPLIT
多卡时每张卡的模型比例。例如 3,1 表示第一张卡分配 75%，第二张卡 25%。
-mg/--main-gpu INDEX
默认: 0LLAMA_ARG_MAIN_GPU
主 GPU 索引。split-mode=none 时选择使用哪张卡；split-mode=row 时用于中间结果和 KV。
-fit/--fit [on|off]
默认: onLLAMA_ARG_FIT
自动调整未设定的参数以适配显存。开启时会根据可用显存自动缩减上下文大小等。
-fitt/--fit-target MiB0,MiB1,...
默认: 1024LLAMA_ARG_FIT_TARGET
--fit 的目标余量（MiB）。每张卡预留多少显存给系统和其他程序。
-fitc/--fit-ctx N
默认: 4096LLAMA_ARG_FIT_CTX
--fit 允许缩减到的最小上下文大小。
-cmoe/--cpu-moe
LLAMA_ARG_CPU_MOE
将 MoE（混合专家）权重全部保留在 CPU。可大幅节省显存，但速度下降。适合 MoE 架构模型（如 Mixtral）。
-ncmoe/--n-cpu-moe N
LLAMA_ARG_N_CPU_MOE
只将前 N 层的 MoE 权重保留在 CPU，其余仍放 GPU。比 --cpu-moe 更灵活。
-ot/--override-tensor <pattern>=<type>,...
LLAMA_ARG_OVERRIDE_TENSOR
覆盖特定张量的缓冲区类型。高级用法，用于精细控制哪些张量放 CPU/GPU。
--rpc SERVERS
LLAMA_ARG_RPC
RPC 服务器列表（host:port），将推理分布到远程机器。

内存管理

--mlock
LLAMA_ARG_MLOCK
强制将模型锁定在 RAM 中，防止被操作系统交换到磁盘。可避免加载后的卡顿，但需要足够的物理内存。
--mmap/--no-mmap
默认: 启用LLAMA_ARG_MMAP
内存映射模型文件。启用时加载更快；禁用时加载慢但可减少 pageout（搭配 --mlock 用不到 mmap）。
-dio/--direct-io
默认: 禁用LLAMA_ARG_DIO
使用 Direct I/O（绕过 OS 页缓存）。仅在特定场景下有益。
--no-host
LLAMA_ARG_NO_HOST
绕过主机缓冲区，允许使用额外的缓冲区。
--repack/--no-repack
默认: 启用LLAMA_ARG_REPACK
是否重新打包权重以优化计算。启用可能提升速度但增加加载时间。
--op-offload/--no-op-offload
默认: 启用
是否将主机张量操作卸载到设备（GPU）执行。
▼

KV Cache

KV cache 通常是第二大显存消耗（仅次于模型权重）。量化 KV 可显著节省显存，代价是微小的质量损失。
-ctk/--cache-type-k TYPE
默认: f16LLAMA_ARG_CACHE_TYPE_K
K 缓存的数据类型。使用更低精度可大幅节省显存。q8_0 是质量和显存的良好折中。
f32f16bf16q8_0q4_0q4_1iq4_nlq5_0q5_1
-ctv/--cache-type-v TYPE
默认: f16LLAMA_ARG_CACHE_TYPE_V
V 缓存的数据类型。同上。K 和 V 可以分别设置不同精度。
f32f16bf16q8_0q4_0q4_1iq4_nlq5_0q5_1
-kvo/--kv-offload/--no-kv-offload
默认: 启用LLAMA_ARG_KV_OFFLOAD
KV 缓存卸载到 GPU。禁用 (--no-kv-offload) 可省显存，KV 会放在 CPU 内存中，但推理速度会降低。
-kvu/--kv-unified
默认: 自动 (slot 数自动时启用)LLAMA_ARG_KV_UNIFIED
统一 KV 缓冲区。所有 slot 共享同一个 KV cache，而不是各占独立的。可提升显存利用率。
-cram/--cache-ram N
默认: 8192 MiBLLAMA_ARG_CACHE_RAM
最大缓存大小（MiB）。-1 = 不限制，0 = 禁用缓存。控制 KV cache 的 RAM 使用上限。
--cache-prompt/--no-cache-prompt
默认: 启用LLAMA_ARG_CACHE_PROMPT
Prompt 缓存。缓存相同前缀的 prompt 计算结果，后续相同前缀的请求会快很多。强烈建议保持启用。
--cache-reuse N
默认: 0LLAMA_ARG_CACHE_REUSE
通过 KV shifting 尝试复用缓存的最小 chunk 大小。需要先启用 --cache-prompt。
--cache-idle-slots/--no-cache-idle-slots
默认: 启用LLAMA_ARG_CACHE_IDLE_SLOTS
新任务到达时保存并清除空闲 slot 的 KV cache。需配合 unified KV 和 cache-ram 使用。
-ctxcp/--ctx-checkpoints N
默认: 32LLAMA_ARG_CTX_CHECKPOINTS
每个 slot 的最大上下文检查点数量。
-cpent/--checkpoint-every-n-tokens N
默认: 8192LLAMA_ARG_CHECKPOINT_EVERY_NT
预填充阶段每处理 N 个 token 创建一个检查点。-1 禁用。
-dt/--defrag-thold N
已弃用
KV cache 碎片整理阈值。已弃用。

注意力与扩展上下文

RoPE 相关参数用于扩展模型的上下文窗口。如果模型本身支持长上下文，通常不需要手动调节。
-fa/--flash-attn [on|off|auto]
默认: autoLLAMA_ARG_FLASH_ATTN
Flash Attention。显著减少显存占用并可能加速推理，尤其在长上下文场景。auto 会自动检测是否支持。
onoffauto
--swa-full
默认: falseLLAMA_ARG_SWA_FULL
使用全尺寸滑动窗口注意力缓存。
--rope-scaling {none,linear,yarn}
LLAMA_ARG_ROPE_SCALING_TYPE
RoPE 频率缩放方法。默认由模型决定。linear 简单线性外推，yarn 更先进的 YaRN 方法。
--rope-scale N
LLAMA_ARG_ROPE_SCALE
RoPE 上下文缩放因子。将上下文扩展 N 倍。例如设 2 则 4K 上下文模型可用 8K。
--rope-freq-base N
默认: 从模型读取LLAMA_ARG_ROPE_FREQ_BASE
RoPE 基础频率，NTK-aware 缩放使用。增大此值可扩展上下文。
--rope-freq-scale N
LLAMA_ARG_ROPE_FREQ_SCALE
RoPE 频率缩放因子。上下文扩展 1/N 倍。
--yarn-orig-ctx N|--yarn-ext-factor N|--yarn-attn-factor N|--yarn-beta-slow N|--yarn-beta-fast N
YaRN（Yet another RoPE eNhancement）的高级参数。通常不需要手动调节，设置 --rope-scaling yarn 后自动生效。
▼

基础采样

采样参数决定模型输出的随机性和多样性。温度越高越随机/有创意，越低越确定/保守。
--temp N/--temperature N
默认: 0.80
温度。控制随机性。0 = 贪婪（总选最高概率），0.7~0.9 适合日常聊天，1.0+ 更随机有创意，2.0 非常随机。
--top-k N
默认: 40LLAMA_ARG_TOP_K
Top-K 采样。只从概率最高的 K 个 token 中选择。0 = 禁用。值越小越保守。
--top-p N
默认: 0.95
Top-P（核采样）。从累积概率达到 P 的最小 token 集合中选择。1.0 = 禁用。通常比 Top-K 效果更好。
--min-p N
默认: 0.05
Min-P 采样。过滤掉概率低于 P × 最高概率 的 token。0.0 = 禁用。是 Top-K/Top-P 的优秀替代方案。
-s/--seed SEED
默认: -1 (随机)
随机种子。-1 = 每次不同。设为固定值可使相同输入产生相同输出（用于调试/复现）。
--ignore-eos
忽略结束符（EOS），强制模型持续生成。相当于设置 --logit-bias EOS-inf。
--samplers SAMPLERS
默认: penalties;dry;top_n_sigma;top_k;typ_p;top_p;min_p;xtc;temperature
采样器执行顺序，用 ; 分隔。可自定义采样管道的执行顺序。
--sampler-seq SEQUENCE
默认: edskypmxt
采样器的简化序列表示。每个字母代表一个采样器。

重复惩罚

用于防止模型陷入重复循环。如果模型经常重复相同内容，尝试增加这些惩罚值。
--repeat-penalty N
默认: 1.00 (禁用)
重复惩罚。大于 1 时惩罚已出现的 token。推荐 1.05~1.2。
--repeat-last-n N
默认: 64
重复惩罚检查窗口大小。0 = 禁用，-1 = 整个上下文。
--presence-penalty N
默认: 0.00 (禁用)
存在惩罚。只要 token 出现过就施加固定惩罚，不管出现几次。鼓励话题多样性。
--frequency-penalty N
默认: 0.00 (禁用)
频率惩罚。惩罚力度与 token 出现次数成正比。比存在惩罚更积极地抑制重复。
--dry-multiplier N|--dry-base N|--dry-allowed-length N
multiplier: 0 (禁用) | base: 1.75 | length: 2
DRY（Don't Repeat Yourself）采样。比传统重复惩罚更智能，专注于惩罚重复的序列模式而非单个 token。--dry-multiplier > 0 时启用。
--dry-sequence-breaker STRING 设定打断重复的序列（默认 \n、:、"、*）。
▼

高级采样

--top-n-sigma N
默认: -1.00 (禁用)
Top-N-Sigma 采样。基于标准差的采样方法，保留均值以上 N 个标准差范围内的 token。
--typical N
默认: 1.00 (禁用)
局部典型采样（Locally Typical Sampling）。基于信息论选择"典型"的 token，而非最高概率的 token。
--xtc-probability N|--xtc-threshold N
probability: 0 (禁用) | threshold: 0.10
XTC（eXtended Token Culling）采样。以一定概率移除超过阈值的 token。
--dynatemp-range N|--dynatemp-exp N
range: 0 (禁用) | exp: 1.00
动态温度。温度在 [temp-range, temp+range] 之间动态变化，当模型"确信"时降温，"不确信"时升温。
--mirostat N|--mirostat-lr N|--mirostat-ent N
mode: 0 (禁用) | lr: 0.10 | ent: 5.00
Mirostat 采样。自适应控制输出"困惑度"。启用后会忽略 Top-K、Top-P 和 Typical 采样。
0 = 禁用，1 = Mirostat v1，2 = Mirostat v2。lr 是学习率，ent 是目标熵。
--adaptive-target N|--adaptive-decay N
target: -1 (禁用) | decay: 0.90
Adaptive-P 采样。选择接近目标概率的 token。decay 控制随时间的适应速率。
-l/--logit-bias TOKEN_ID(+/-)BIAS
手动调整特定 token 的生成概率。+ 增加概率，- 降低概率。例如 15043+1 增加 token 15043 的概率。
-bs/--backend-sampling
默认: 禁用实验性
在后端（GPU）执行采样。可减少 CPU-GPU 数据传输开销。
▼

约束输出

用于强制模型按特定格式（如 JSON）输出。
--grammar GRAMMAR
使用 BNF-like 语法约束输出格式。参见 grammars/ 目录中的示例。
--grammar-file FNAME
从文件读取 BNF 语法。
-j/--json-schema SCHEMA
使用 JSON Schema 约束输出。{} 表示任意 JSON 对象。
-jf/--json-schema-file FILE
从文件读取 JSON Schema。

推理/思考模式

用于控制支持"思考"的模型（如 DeepSeek-R1）的推理行为。
-rea/--reasoning [on|off|auto]
默认: autoLLAMA_ARG_REASONING
思考模式开关。auto 从模板自动检测。
on 强制开启off 强制关闭auto 自动检测
--reasoning-format FORMAT
默认: autoLLAMA_ARG_THINK
思考内容的输出格式。
none 不解析思考标签 deepseek 放入 reasoning_content deepseek-legacy 保留 <think> 标签且填充 reasoning_content
--reasoning-budget N
默认: -1 (不限)LLAMA_ARG_THINK_BUDGET
思考预算（token 数）。-1 = 不限制，0 = 立即结束思考，N > 0 = 最多思考 N 个 token。
--reasoning-budget-message MESSAGE
LLAMA_ARG_THINK_BUDGET_MESSAGE
当思考预算耗尽时，在结束思考标签前注入的提示消息。

网络与监听

--host HOST
默认: 127.0.0.1LLAMA_ARG_HOST
监听地址。127.0.0.1 = 仅本机访问，0.0.0.0 = 局域网可访问。也支持 Unix socket（地址以 .sock 结尾）。
--port PORT
默认: 8080LLAMA_ARG_PORT
监听端口。如端口被占用需更换。
--reuse-port
默认: 禁用LLAMA_ARG_REUSE_PORT
允许多个 socket 绑定同一端口。
-to/--timeout N
默认: 600 秒LLAMA_ARG_TIMEOUT
服务器读写超时时间（秒）。
--ssl-key-file FNAME|--ssl-cert-file FNAME
SSL/TLS 证书和私钥文件路径，用于 HTTPS。
--path PATH
LLAMA_ARG_STATIC_PATH
提供静态文件服务的目录路径。
--api-prefix PREFIX
LLAMA_ARG_API_PREFIX
API 路径前缀（不含尾斜杠）。例如 /v1。
--sleep-idle-seconds SECONDS
默认: -1 (禁用)
空闲自动休眠。指定空闲多少秒后服务器进入休眠状态。

LLAMA_API_KEY
API 密钥。启用后所有请求需携带此 key 才能访问。支持多个 key，逗号分隔。
--api-key-file FNAME
从文件读取 API 密钥。

端点控制

--metrics
默认: 禁用LLAMA_ARG_ENDPOINT_METRICS
启用 Prometheus 兼容的 metrics 端点（/metrics）。用于监控。
--props
默认: 禁用LLAMA_ARG_ENDPOINT_PROPS
允许通过 POST /props 修改全局属性。
--slots/--no-slots
默认: 启用LLAMA_ARG_ENDPOINT_SLOTS
暴露 slot 监控端点（/slots）。可查看每个 slot 的状态。
--slot-save-path PATH
默认: 禁用
slot KV cache 的保存路径。启用后可持久化 slot 状态。
--embedding
默认: 禁用LLAMA_ARG_EMBEDDINGS
启用 embedding 功能。仅用于专用的嵌入模型。
--rerank
默认: 禁用LLAMA_ARG_RERANKING
启用 reranking 端点。
