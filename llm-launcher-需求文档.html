LLM 本地模型启动器

需求文档
0%

一个简易桌面应用，用于可视化加载本地 GGUF 模型、自定义运行参数、后台检测与进程管理。

开发策略：阶梯制，每阶段确认可行后再进入下一阶段。
▶ 第一阶段：核心启动控制（MVP）
已完成

替代现有的 .bat 菜单脚本，实现可视化参数配置 + 一键启动/停止。

1.1 模型选择 4/4
  • 浏览本地文件选择 .gguf 主模型（FileBrowser 弹窗）
  • 浏览本地文件选择 mmproj 视觉投影文件（可选，独立浏览按钮）
  • 记住上次选择的路径（ConfigStore 写入 config.yaml）
  • 自动检测同目录下的 mmproj（文件名含 mmproj 且为 .gguf）
1.2 核心参数配置 5/5
  • 上下文大小（-c）：下拉 2048 / 4096 / 8192 / 16384 / 32768
  • GPU 层数（--n-gpu-layers）：数字输入 + 「全GPU」(9999) / 「CPU」(0) 快捷按钮
  • 并发数（-np）：下拉 1 / 2 / 4 / 8
  • 端口（--port）：数字输入框，默认 8080，冲突时提示
  • 监听地址（--host）：下拉 127.0.0.1（仅本机）/ 0.0.0.0（局域网）
1.3 服务器生命周期 4/4
  • 启动按钮：调用 ProcessSupervisor.start() 启动 llama-server
  • 停止按钮：SIGTERM → 等 3s → SIGKILL
  • 状态指示：运行中（绿）/ 已停止（灰）/ 启动中（黄）/ 异常退出（红）
  • 启动后自动打开浏览器（auto_open_browser Switch 开关）
1.4 进程存活检测 3/3
  • 实时检测 llama-server 进程是否在运行（_poll_loop 每 1 秒轮询 PID）
  • 进程意外退出时弹通知 + 自动回收到"已停止"状态
  • 端口被占用时提示用户换端口（PortInUseError 捕获）
1.5 基础日志 3/3
  • 显示 llama-server 的标准输出/错误输出（只读，RichLog 面板）
  • 日志自动滚动到最新行
  • 关键事件高亮（加载完成 / 错误 / OOM，关键词染色）

验收标准

  • 能通过 GUI 选择模型文件、调整 5 个核心参数、启动/停止服务器
  • 启动后浏览器能正常访问 Web UI 和 API
  • 进程异常退出时 UI 状态正确更新
▶ 第二阶段：高级参数与运行时监控
已完成

覆盖更多 llama.cpp 参数，加入实时资源监控。

2.1 高级参数 28/28
KV Cache 与显存
  • KV Cache 量化类型（-ctk / -ctv）：下拉 f16 / q8_0 / q4_0
  • 统一 KV 池（-kvu）：Switch 开关，多 slot 共享 KV
  • KV 不放 GPU（--no-kv-offload）：Switch 开关
  • Flash Attention（-fa）：Switch 开关
  • Prompt Cache（--cache-prompt）：Switch 开关
  • 空闲 Slot 复活（--cache-idle-slots）：Switch 开关
  • Cache RAM 上限（--cache-ram）：数字输入，默认 8192 MiB
推理速度
  • 线程数（-t）：数字输入，默认 -1（CPU 核心数）
  • Prompt 线程数（-tb）：数字输入，默认等于 -t
  • 逻辑批大小（-b）：数字输入，默认 2048
  • 物理批大小（-ub）：数字输入，默认 512
  • HTTP 线程数（--threads-http）：数字输入，默认 -1
  • 跳过预热（--no-warmup）：Switch 开关
采样参数
  • 温度（--temp）：滑块 0.0 ~ 2.0，默认 0.80
  • Top-K（--top-k）：数字输入，默认 40
  • Top-P（--top-p）：滑块 0.0 ~ 1.0，默认 0.95
  • Min-P（--min-p）：滑块 0.0 ~ 1.0，默认 0.05
  • 重复惩罚（--repeat-penalty）：滑块 0.5 ~ 2.0，默认 1.0
  • 随机种子（-s）：数字输入，-1 为随机
  • 最大生成长度（-n）：数字输入，-1 为无限
  • 忽略 EOS（--ignore-eos）：Switch 开关
思考/推理模式
  • 思考模式（-rea）：下拉 on / off / auto
  • 思考格式（--reasoning-format）：下拉 none / deepseek / deepseek-legacy
  • 思考预算（--reasoning-budget）：数字输入，-1 不限，0 不思考
多模态
  • 视觉编码器放 GPU（--mmproj-offload）：Switch 开关
  • 最小视觉 Token（--image-min-tokens）：数字输入
  • 最大视觉 Token（--image-max-tokens）：数字输入
安全与访问控制
  • API Key（--api-key）：文本输入，留空不传参
  • 超时秒数（--timeout）：数字输入，默认 600
  • Prometheus 监控（--metrics）：Switch 开关
  • Slots 端点（--slots）：Switch 开关
2.2 运行时监控 4/4
  • GPU 利用率 / 显存占用（调用 nvidia-smi）已验证
  • CPU 占用：系统总体 + 进程级（归一化到全核平均）已验证
  • 内存占用：系统已用/总量 + 进程 RSS（类似任务管理器）已验证
  • 启动时输出完整命令到日志面板（每参数一行）已验证
2.3 预设管理 0/4
  • 保存当前参数为预设（命名 + 存储）
  • 预设列表，一键切换
  • 预设导入/导出（JSON 文件）
  • 覆盖预设中的已有名称时二次确认
2.4 API 测试面板 0/2
  • 内建简易聊天框，直接测试 API（纯文本）
  • 支持粘贴图片 URL 测试多模态

验收标准

  • 高级参数全部可用，与 llama.cpp 命令行效果一致 已验证
  • 参数分组折叠显示，默认展开常用组 已验证
