P0 强烈推荐

10 项
--fit [on|off] 默认: on
自动适配显存。显存不够时自动缩减上下文大小等参数，避免 OOM 崩溃。
新手必备，启用后不会因参数设太大而启动失败
onoff
Switch 开关，默认开启
GPU 与显存 组
--fit-target MiB 默认: 1024
搭配 --fit，设定每张卡预留多少 MiB 给系统和其他程序。
跑模型同时开浏览器/游戏时，调大此值避免显存抢占
数字输入，仅当 fit=on 时可见
GPU 与显存 组
--mlock 默认: 禁用
锁定模型到内存，防止 OS 将模型数据交换到磁盘。
CPU 推理或混合推理时，避免加载后因 pagefault 产生卡顿
Switch 开关
内存管理 组（可放入 KV Cache 与显存）
--repeat-last-n N 默认: 64
重复惩罚检查窗口。当前已有 --repeat-penalty 却缺少此配套参数，窗口过小惩罚效果弱。
完善已有的重复惩罚功能，0=禁用，-1=整个上下文
数字输入
采样参数 组（紧跟 repeat-penalty 后面）
--presence-penalty N 默认: 0.00 (禁用)
存在惩罚。只要 token 出现过就施加固定惩罚，不管出现几次。鼓励话题多样性。
OpenAI 兼容 API 标准参数，前端应用经常传此字段
滑块 0.0 ~ 2.0
采样参数 组
--frequency-penalty N 默认: 0.00 (禁用)
频率惩罚。惩罚力度与出现次数成正比，比 presence 更积极抑制高频重复。
与 presence-penalty 同为 OpenAI API 标准字段
滑块 0.0 ~ 2.0
采样参数 组
--dynatemp-range N 默认: 0.00 (禁用)
动态温度。温度在 [temp−range, temp+range] 之间浮动：模型确信时自动降温，不确信时升温。
社区热门采样方式，比固定温度效果更自然
滑块 0.0 ~ 1.0，设 0 禁用
采样参数 组（紧跟 temperature 后面）
--chat-template NAME 默认: 从模型读取
聊天模板。覆盖模型内嵌的对话格式。当模型自带模板不对或需要切换时使用。
日常排错第一高频需求："模型输出格式乱了"→ 换模板解决
chatmlllama3deepseek3minicpmgemmaphi4mistral-v7…
下拉选择（内置模板列表）+ 可选文本输入自定义
新增「聊天模板」组，或放入「安全与访问控制」
--context-shift / --no-context-shift 默认: 禁用
上下文移位。上下文写满时自动丢弃早期内容继续生成，而非报错截断。
长对话/无限续写场景刚需，不开的话上下文满了就停了
Switch 开关
核心参数 组（靠近 context_size）
--sleep-idle-seconds N 默认: -1 (禁用)
空闲休眠。服务器无请求时经过 N 秒自动休眠，释放 GPU 资源。
挂机用户省电省显存，尤其笔记本/共享 GPU 场景
数字输入，-1 禁用
服务器 组（或新增「自动化」组）

P1 建议添加

12 项
-a / --alias STRING 默认: 空
模型别名。设置 API 返回的模型名称（OpenAI 兼容 API 的 model 字段）。
接 Open WebUI 等前端时，用别名区分不同模型/配置
文本输入
安全与访问控制 组
--cache-reuse N 默认: 0
通过 KV shifting 复用缓存的最小 chunk 大小，需配合 --cache-prompt。
多轮对话时加速，已有 cache-prompt 但缺少此进阶控制
数字输入
KV Cache 与显存 组
--mirostat N 默认: 0 (禁用)
Mirostat 采样。自适应控制输出"困惑度"，启用后忽略 Top-K/Top-P/Typical。
长文/创意写作用户群体大，文学创作社区常用
0 禁用1 Mirostat v12 Mirostat v2
下拉选择 0/1/2
采样参数 组
--dry-multiplier N 默认: 0.00 (禁用)
DRY 采样。惩罚重复的序列模式（不只是单 token），比 repeat-penalty 更智能。
防重复的最新方案，效果优于传统惩罚
滑块 0.0 ~ 3.0，设 0 禁用
采样参数 组
--lora FNAME 默认: 空
加载 LoRA 适配器。虽然目标用户不做微调，但使用别人训好的 LoRA（角色/风格/能力增强）很常见。
角色扮演/特定领域能力增强场景，用户只需下载 .gguf LoRA 文件
文件浏览器选择（同主模型），可多选
新增「LoRA 适配器」组，或放入模型选择区域
-sm / --split-mode {none,layer,row} 默认: layer
多 GPU 拆分策略。
双卡用户越来越多，选错模式性能差距很大
none 仅用一张卡layer 按层拆分row 按行并行
下拉选择
GPU 与显存 组（紧跟 n-gpu-layers 后）
-ts / --tensor-split N0,N1,... 默认: 空 (均分)
多卡比例分配。如 3,1 表示第一张卡 75%，第二张卡 25%。
不同显存大小的双卡必须设，否则小卡 OOM
文本输入（逗号分隔数字），仅 split-mode≠none 时可见
GPU 与显存 组
--jinja / --no-jinja 默认: 启用
Jinja 模板引擎开关。极少数模型的内嵌模板与 Jinja 引擎不兼容，需关闭排错。
与 --chat-template 配合，模板排错的完整工具链
Switch 开关
跟随 chat-template
--ui / --no-ui 默认: 启用
关闭内置 Web UI，仅保留 API 端点。
只做 API 后端时关闭省资源，避免暴露 UI 页面
Switch 开关
安全与访问控制 组
--log-file FNAME 默认: 空 (标准输出)
日志输出到文件而非终端。
出问题后回溯排查，尤其服务器长时间运行场景
文件路径输入（或文件浏览器）
新增「日志」组，或放入杂项
--prefill-assistant / --no-prefill-assistant 默认: 启用
最后一条消息是 assistant 时，是否将其作为续写前缀而非完整消息。
角色扮演/辅助写作时的关键行为控制
Switch 开关
跟随 chat-template
-sps / --slot-prompt-similarity N 默认: 0.10
请求 prompt 需与 slot 已有 prompt 匹配到什么程度才复用该 slot。0 = 禁用。
多人共享一个服务时优化 slot 复用效率
滑块 0.0 ~ 1.0
KV Cache 与显存 组
